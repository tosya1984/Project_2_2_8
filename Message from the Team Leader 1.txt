Повідомлення від Тім Ліда (Олег Петров):

Огляд проекту:
Розробка платформи для обліку запасів, яка дозволяє користувачам відстежувати запаси товарів, переглядати історію руху товарів і прогнозувати потреби в поповненні. Платформа повинна мати надійний бекенд для управління даними про запаси, API для інтеграції з іншими системами, а також статичний фронтенд для зручного користування.

Задача:
Розгорнути інфраструктуру з бекендом для обробки запитів, забезпечити кешування для прискорення доступу до часто запитуваних даних, а також налаштувати фронтенд через S3 та CloudFront для швидкого завантаження контенту. При виконанні завдання студенти мають спиратись на надану архітектурну діаграму для належного налаштування всіх компонентів системи в межах бюджету $160 на місяць.

База даних:
RDS (PostgreSQL) використовується для зберігання інформації про запаси та історію операцій. Redis застосовується для кешування, що підвищує продуктивність системи при доступі до частих запитів.

Докеризація:
Усі сервіси мають бути докеризовані для забезпечення легкої масштабованості та зручного управління інфраструктурою. Це дозволить швидко розгортати нові функції і спростить підтримку системи.

CI/CD процес:
Налаштувати CI/CD процес для автоматизації деплою додатку, що забезпечить стабільність роботи платформи і швидке впровадження оновлень.

Важливі нотатки:
Платформа повинна підтримувати обробку 5 запитів за секунду у звичайних умовах та до 10 запитів під час пікових періодів. Загальний бюджет для інфраструктури не повинен перевищувати $160 на місяць.
